01 / 08

How an individual lives inside Align — every day.

The daily interaction loop is not a rigid schedule. It is a set of lightweight touchpoints that collectively update the User Model, surface relevant intelligence, and deepen the AI companion's context without requiring significant effort from the user. Every touchpoint is optional. Every touchpoint matters.

◎
MORNING · 6–9AM · OPTIONAL · 60 SECONDS
B3 Daily Check-in
9 questions across Biological, Psychological, and Spiritual domains. Tap interface — one question visible at a time. Completes automatically on the 9th answer. Produces A/B/C Day classification. Master Chief loads this before the first conversation of the day.
PROMPTMaster Chief surfaces proactively if B3 not yet completed and user opens the app before 10am. One tap to begin — never a modal interrupt.
MODEL UPDATEB3 state written to rhythmOS.currentDayState. Affects AI persona pacing for entire day. Suppresses high-stakes prompts on C Days.
✦
MORNING · FIRST OPEN · CONTEXT-AWARE
Master Chief morning signal
Master Chief pre-loads the full PersonaContext — User Model, B3 state, season, recent Conviction Events, active decisions, Life Journey entries — and surfaces one morning signal calibrated to the day. Not a notification. A framing. The first thing the person sees when they open Align.
PROMPTAuto-generated from PersonaContext. A Day: surfaces Release Threshold practice or active decision. B Day: surfaces one moderate recommendation. C Day: surfaces rest, recovery, or support signal. Never urgent framing.
PASSIVEUser can dismiss or respond. If they respond, opens full Master Chief conversation with full context already loaded.
◈
ANYTIME · ACTIVE SESSIONS · VARIABLE LENGTH
Master Chief conversations
User initiates. Master Chief has already loaded the full PersonaContext before the conversation begins. Every response is calibrated to wiring profile, current season, B3 state, primary gap, active decisions, and recent Conviction Events. The conversation feels like talking to someone who already knows you — because the system does.
MODEL UPDATEConversation signals analyzed in background. Conviction Event detector runs. Decision resolutions, boundary narratives, lesson statements flagged for Life Journey logging prompt at conversation end.
PROMPTAt conversation end: if Life Journey logging signals detected, system surfaces bottom-sheet prompt. Never modal. Always optional. "Something worth capturing happened — want to log it?" User selects: Conviction Event / Lesson / Life Event / Not now.
◇
ANYTIME · 2–5 MINUTES · HIGH MODEL VALUE
Journal, voice note, or quick capture
User writes, speaks, or types what is happening — a decision, a hard moment, a win, a lesson. The Narrative Layer™ receives it. AI processes for semantic signals: Conviction Event patterns, season language, belonging signals, purpose connection. Updates propagate to User Model based on detected signals.
MODEL UPDATENarrative Layer stores the entry. AI extracts: entity type (person/org/decision), emotional valence, temporal marker (season), domain relevance (which of 5 IR domains affected). Sub-score update flags generated where thresholds met.
TRIGGERIf audio: transcribed via speech-to-text API, then processed identically to text journal. Audio stored in Narrative Layer alongside transcript. Master Chief can reference "what you said in the voice note on Tuesday" in future sessions.
○
EVENING · OPTIONAL · 3–5 MINUTES
Decision Suite or active product session
User brings a real decision, a career navigation question, a design challenge, or a financial scenario. The product reads from the current User Model state. Output is calibrated to their wiring, season, and B3 state. Not generic advice. The system knows who is asking.
MODEL UPDATEDecision resolution — if user confirms a decision made — flags as Conviction Event candidate. System asks: "Want to log this as a Conviction Event? It will update your Release Threshold and Opportunity Readiness scores."
PASSIVEAll product sessions logged to interaction history. Frequency and type of product use feeds engagement signal. Surfaces in coach portal if coach context active.
◉
BACKGROUND · ALWAYS · ZERO USER EFFORT
Passive signal intake
Wearable data (HRV, sleep), calendar density (meeting load), and connected integrations (Notion tasks, Slack metadata) feed into RhythmOS continuously. No user action required. B3 classification becomes more accurate as passive signals accumulate. The model improves while the user sleeps.
PASSIVEAll passive signals require explicit user consent during onboarding. Each integration is independently revocable. None produce outbound data from Align to the connected service.
MODEL UPDATERhythmOS re-classifies B3 day type each morning using accumulated signals. rhythmOS.confidence increases as passive signals accumulate. Low confidence = B3 shown as tentative in UI.
02 / 08

How Align checks in and works with people.

Align uses a structured prompt architecture — not push notifications. Every system-initiated touchpoint is calibrated to the individual's User Model state, B3 classification, and active context. None are urgent. None are generic. Each is produced from a specific trigger condition.

INDIVIDUAL · DAILY · MORNING
B3-calibrated morning frame
"Today is an A Day. Full capacity across all three domains. Your Release Threshold practice window is open. The partnership decision is in your queue. This is the moment."
Trigger: First app open after B3 check-in completion, B3 state = A, active decision in Decision Suite, primary gap = Release Threshold
INDIVIDUAL · WEEKLY · SEASON CHECK
Season alignment check
"You have been in Building season for 6 weeks. Master Chief: the Organizer runs the diagnostic continuously past the point where the evaluation is useful. You named what done looks like. Has it arrived?"
Trigger: 6 weeks in same season without Life Journey season transition event, wiring = Organizer, Commitment Depth = 7
INDIVIDUAL · EVENT-DRIVEN · DECISION
Pre-delivery coaching
"You are about to send the offer. Before you do: does it have a qualifier in it? A softener before they push back? State it. Then stop. They will respond."
Trigger: User opens Decision Suite, primary gap = Release Threshold ≤ 3, decision type = partnership/pricing/offer
INDIVIDUAL · EVENT-DRIVEN · MILESTONE
Conviction Event detection
"Something significant just happened in your conversation. Want to capture it? This looks like a Conviction Event — it would update your Capability domain. One tap to log."
Trigger: Conversation end, Conviction Event detector fires (decision resolution, boundary narrative, or achievement mention detected)
ENTERPRISE · WEEKLY · ADMIN
Org capacity signal summary
"This week: 7 of 12 team members at high capacity. Expansion season signals emerging in leadership. Q3 initiative window is open. Strategic Alignment Score: 74. One capacity gap identified."
Trigger: Weekly org signal computation complete, admin has Strategic Alignment Score enabled, min 5 consented members in aggregation pool
INDIVIDUAL · CONTEXTUAL · AI-ERA
AI-Era disruption signal
"A significant AI capability shift dropped this week. Based on your AI-Era Readiness (94%) and your Skill Confidence (10/10): here is where this amplifies your edge rather than threatens it."
Trigger: Significant AI news event detected via API, user has AI-Era Readiness score on file, score ≥ 70 (surfaces amplification angle, not threat framing)

CRITICAL RULE · SUB-SCORE UPDATE LOGIC
B3 check-ins and passive signals never directly modify assessment sub-scores. Only Conviction Events (user-confirmed) and formal reassessments update the IR/VS scoring layer. B3 and passive signals update the pacing and context layers only. This distinction must be enforced at the API layer.
04 / 08

The Narrative Layer™

Every other layer of the User Model is quantitative. The Narrative Layer is where meaning lives. It stores what cannot be scored — and feeds back into every product, every AI conversation, and every coaching session.

QUANTITATIVE LAYER — Scores, metrics, readiness signals
Conviction Score™ — 0 to 100
Value Score™ — 0 to 100
Sub-scores — 0, 3, 7, or 10
AI-Era Readiness — composite %
B3 Day classification — A / B / C
Domain scores — Identity, Capability, etc.
Growth velocity — score change / day
Wiring profile — type weights
NARRATIVE LAYER™ — Story, meaning, turning points
Life events — named, dated, typed
Lessons — extracted from journals and conversations
Turning points — season transitions, major decisions
Milestones — completed, logged, celebrated
Wins — behavioral evidence of growth
Failures — reframed as source material
Relationships — named people, type, impact
Season story — what this season is building toward
// NarrativeEntry schema — stored in Life Journey, feeds PersonaContext
NarrativeEntry {
  entryId: UUID
  userId: UUID
  createdAt: timestamp
  entryType: LifeEvent | Lesson | TurningPoint | Milestone | Win | Failure | Relationship
  source: Journal | VoiceNote | AIConversation | ManualEntry | CoachSession
  content: string // The raw text or transcript
  aiExtractedSignals: {
    convictionEventCandidate: boolean
    affectedDomain: Domain | null
    seasonLanguageDetected: boolean
    belongingSignal: boolean
    purposeSignal: boolean
  }
  userConfirmed: boolean // Conviction Event only applied if confirmed
  season: Season // Season active at time of entry
  personaContextFeed: boolean // Whether MC loads this in PersonaContext
}
WHY THE NARRATIVE LAYER EXISTS
Humans think in stories, not scores. A person who reads their Conviction Score of 86 processes it intellectually. A person who reads "the moment you sent the offer without the qualifier, and they said yes without pushing back" processes it emotionally. The Narrative Layer™ stores the behavioral evidence that makes scores meaningful — and gives Master Chief the specific references that make responses feel like they come from someone who was in the room. Quantitative without narrative is a dashboard. Narrative with quantitative is a companion.
05 / 08

How enterprises interact with Align daily.

Enterprise interaction is governed by the semantic bridge architecture. Individual data never crosses. What crosses is meaning. The enterprise's daily interaction with Align is at the aggregated organizational signal level — never at the individual level.

ENTERPRISE · ADMIN · DAILY DRIVER
Strategic Alignment Score™ dashboard
Admin opens StrategyOS each morning. Sees the org-level Strategic Alignment Score (0–100), season distribution across the team, capacity signal (High/Moderate/Low), and any initiative flags. Master Chief surfaces one org-level signal calibrated to the admin's own User Model and the org's current state.
Score computed from semantic signals only — never from individual sub-scores
Min 5 consented members for aggregation. Below 5: preliminary signal only with caveat flag
Admin's own Master Chief reads both their personal User Model AND the org signal layer
No individual is identifiable in any enterprise view — even to the admin
ENTERPRISE · WEEKLY · STRATEGIC
Capacity Planning Intelligence™ prompt
StrategyOS generates a weekly prompt to the admin synthesizing team capacity signals against active strategic initiatives. Surfaces mismatches between what the strategy demands and what the team's current season and readiness signals can support. Gives a specific recommendation, not a report.
Prompt generated from: org capacity signal + active initiatives + Strategic Alignment Score trend
Recommendation framed as strategic timing, not people assessment
Admin can bring prompt into their own Master Chief conversation for deeper exploration
All recommendations logged to org Decision Suite for tracking
ENTERPRISE · INDIVIDUAL MEMBER · DAILY
Individual inside the enterprise context
From the individual's perspective, the enterprise context is just a context badge — it adds org season signals to their feed and allows their meaning profile (consented tier) to contribute to the org aggregate. Their daily Align experience is unchanged. Their privacy is unchanged.
Individual's User Model is unaffected by enterprise context activation
Enterprise context adds signals to the individual's feed — never removes or overrides
Individual can disconnect the enterprise context at any time — immediate and atomic
What they see in their context feed: org season, team capacity, relevant strategic signals
ENTERPRISE · COACH PORTAL · SESSION-DRIVEN
Coach as enterprise interface
For enterprise accounts with coach portals active, coaches interact with Align through a session-prep interface that surfaces semantic signals about their clients — never raw scores without consent. The coach's own Master Chief also reads from their personal User Model, making them a better coach while using the platform.
Coach sees: session preparation signals, client season, capacity signal (if consented), recent Narrative Layer themes (if shared)
Coach never sees: assessment scores, sub-scores, B3 state, AI conversations
Coach can log session notes → AI extracts signals → flags for client confirmation
Coach portal white-label available — coach can brand as their own product

API architecture overview.

How the layers connect. All external APIs are intake-only — data flows into the User Model, never out. The semantic bridge enforces this at the infrastructure layer.

// External API intake flow — all opt-in, all intake-only

// Layer 2 — Wellness intake
WellnessIntake.receive(source: AppleHealth | Oura | Garmin) → RhythmOS.calibrate()

// Layer 3 — Behavioral intake
BehavioralIntake.receive(source: GoogleCalendar | Notion | Asana) → ConvictionEventDetector.scan()

// Layer 4 — Health intake (DiagnosticOS)
HealthIntake.receive(source: FHIR | LabResult | PractitionerNote) → DiagnosticOS.process()

// Layer 5 — Market intake (StrategyOS)
MarketIntake.receive(source: NewsAPI | EDGAR | CrunchBase) → StrategyOS.crossReference()

// Layer 6 — Agent output (writes user data, reads User Model as context)
AgentLayer.loadContext(userId) → PersonaContext → Agent.execute(task)

// HARD RULE — enforced at API gateway, not application code
UserModel.export() → BLOCKED // User Model never leaves Align systems
SemanticBridge.pass(rawScore) → BLOCKED // Scores never cross to enterprise
SemanticBridge.pass(meaning) → ALLOWED // Meaning crosses with consent
08 / 08

Non-negotiable interaction rules.

These rules govern every interaction surface — individual, enterprise, coach, and agent. They are not guidelines. They are architectural constraints that must be enforced at the API and data layer.

R1
Master Chief is context-aware, not query-aware
The full PersonaContext must be loaded before every conversation begins. A response generated without PersonaContext is not an Align response. There is no "quick reply" mode that bypasses context loading.
PersonaContext.load(userId) → REQUIRED before claude.messages.create()
R2
Prompts are calibrated, never broadcast
No system-initiated prompt is generic. Every prompt must derive from at least one specific User Model signal. A prompt that would be identical for any user is not an Align prompt.
Prompt.generate(template, userSignals) → must include ≥1 user-specific signal
R3
B3 check-ins never update assessment sub-scores
B3 state updates only rhythmOS.currentDayState and the pacing layer. It never writes to IR or VS sub-scores. These are separate data namespaces and must remain so at the schema level.
B3State → rhythmOS namespace only. Never → layer2.impactReadiness.domains
R4
Conviction Events require user confirmation
The system can detect, flag, and surface Conviction Event candidates. It cannot apply a sub-score update without explicit user confirmation. Detection and application are two separate steps with user action required between them.
ConvictionEvent.detect() → flag. ConvictionEvent.apply() → requires userConfirmed === true
R5
Life Journey logging prompt appears at conversation end, never beginning
The logging prompt is a reward for a meaningful conversation, not a pre-commitment. It surfaces as a bottom sheet — never a modal. It is dismissable without friction. "Not now" is always available and never followed by a repeat prompt in the same session.
LifeJourney.promptTiming === 'conversationEnd'. Modal === false. RepeatInSession === false
R6
Enterprise aggregation minimum: 5 members
No aggregate signal is computed or surfaced for groups below 5 consented members. Below 5: return isPreliminary: true with explicit UI caveat. Below 2: return null.
aggregateOrgSignals(members) → if members.length < 2: return null. if < 5: isPreliminary: true
R7
C Day suppression is non-negotiable
When B3 classification is C Day, all high-stakes prompts (Release Threshold, Ask Capacity, partnership decisions, visibility challenges) are suppressed from proactive surfacing. The suppression list is maintained as a constant and is not overridden by any product logic.
if (B3State === 'C') { filterPrompts(HIGHSTAKES_SUPPRESSION_LIST); }
R8
All external API intake is inbound-only
No external API integration produces outbound data from Align to the connected service. Apple Health, Google Calendar, Oura, Notion — all receive zero data from Align. The semantic bridge guard applies to all API connections, not just enterprise outputs.
ExternalAPI.request().method === 'GET' | 'READ' only. POST to external → BLOCKED at gateway
THE NORTH STAR FOR EVERY INTERACTION DECISION
"Does this interaction make the User Model more accurate — or does it make the person feel seen by what the User Model already knows? If the answer to either is yes, ship it."
